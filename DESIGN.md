# DESIGN — e-Stat アプリケーション ID 取得案内 UI（Phase 3.6）

**Sprint**: Phase 3.6
**Date**: 2026-04-02
**Status**: 完了（Ship 済み）

---

## 背景・課題

e-Stat の検索機能を使うにはアプリケーション ID が必要だが、
設定ページ（`CredentialsForm`）と未設定バナー（`EStatBanner`）には
入力欄のみが存在し、ID の取得方法が案内されていない。
初回ユーザーは公式サイトを自力で探す必要があり、離脱につながっている。

---

## 目標・非目標

### 目標

- `CredentialsForm`（設定ページ）に番号付きの取得手順セクションを追加する
- `EStatBanner` に取得案内リンクを追加する
- e-Stat 公式 URL は定数で管理し、変更時に一箇所で修正できるようにする

### 非目標

- 手順内容の自動更新・スクレイピング
- e-Stat 公式サイトの手順変更の検知
- 多言語対応（英語など）

---

## 制約条件

- e-Stat 公式サイトの URL・手順は外部コンテンツのため陳腐化リスクがある
  → URL・手順テキストを定数ファイルにまとめ、更新箇所を最小化する
- 既存の `CredentialsForm` および `EStatBanner` のレイアウトを大きく変えない

---

## 成功指標

- [ ] 設定ページに番号付き取得手順（3〜4 ステップ）が表示される
- [ ] e-Stat バナーに「ID の取得方法はこちら」リンクが表示される
- [ ] e-Stat 公式 URL が定数で管理されている
- [ ] 既存テストが全通過する
- [ ] 追加コンポーネントのテストが追加されている

---

## リスクと前提

### R1: e-Stat 公式サイトの仕様変更（最重要）

手順の URL や画面が変わると案内が古くなる。

**対策**: URL・手順テキストを `lib/estat-guide.ts`（仮）などの定数ファイルに集約し、
メンテナンスコストを最小化する。必要に応じてコードコメントに「最終確認日」を記載する。
